Overriding Specificity !

In this lesson we are learning how to override the specificity in a way that it can never get overridden again.

Note : This heading is styled as Red color with cyan background in the CSS file but the style will not get updated here despite using ID selector, along with chaining it to the class selector

We can override it in two ways :

  • Inline Element : They have specificity value of 1000
  • !Important Tag :

The CSS Applied on each of them !
  1. This is the first CSS that was applied and this was overridden by the second one. h1 { background-color: cyan; color: red;}
  2. This is the second CSS that was applied and this was overridden by the third one. .main-heading { background-color: yellow; color: rgb(241, 89, 152); } /* this is class selector and this overrides the element selector */
  3. This is the third CSS that was applied and this was overridden by the fourth one. #main-headblock { background-color: darksalmon; color: rgb(0, 0, 0); } /* this is id selector and it overrides the class selector */
  4. This is the fourth CSS that was applied and this was overridden by the fifth one. #main-headblock.main-heading { background-color: green; color: red; } /* this is chaining selectors to increase the specificity and this will be applied */
  5. This is the fifth CSS that was applied and this was overridden by the second one. html body main section .main-heading#main-headblock { background-color: red; color: green; } /* and this is another way to increase the specificity */

Even this can be overridden by inline styling or by !important keyword.

Using !IMPORTANT on 1 that is the lowest in specificity and IMPORTANT CAN BE OVERRIDDEN BY IMPORTANT.
